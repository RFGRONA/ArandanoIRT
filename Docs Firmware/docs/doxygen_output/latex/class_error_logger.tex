\doxysubsection{Detailed Description}
Provides a static method to format and send error details as JSON via HTTP POST. 

This utility class simplifies sending structured error messages to a backend endpoint. It includes checks for network connectivity and backend status before attempting to send.

Sends an error log message to the specified \doxylink{class_a_p_i}{API} endpoint. 

Static method implementation for sending structured error logs.

Constructs a JSON payload with the error source and message\+: {\ttfamily \{"{}source"{}\+: error\+Source, "{}message"{}\+: error\+Message\}} and sends it via HTTP POST to the given URL.

\begin{DoxyReturn}{Returns}
{\ttfamily true} if the log message was successfully sent AND the server responded with an HTTP 2xx status code. Returns {\ttfamily false} if Wi\+Fi is disconnected, if the backend status check (via the global {\ttfamily api} object) fails, if the HTTP request fails, or if the server responds with a non-\/2xx status code. 
\end{DoxyReturn}
\begin{DoxyNote}{Note}
This function depends on a globally available {\ttfamily api} object of class {\ttfamily \doxylink{class_a_p_i}{API}} to check backend status via {\ttfamily api.\+check\+Backend\+Status()}. Ensure this object is instantiated globally. 

This function currently uses an HTTP connection ({\ttfamily Wi\+Fi\+Client}). If the {\ttfamily api\+Url} requires HTTPS, the implementation in the .cpp file must be modified to use {\ttfamily Wi\+Fi\+Client\+Secure} and handle certificate validation appropriately.
